bug in my original global_deny.conf script.

I just wanted to send a quick note to say that my previous post detailing how to create a global IP deny list for Apache had an incorrect specification of the “Order Allow,Deny” line.  I’ve corrected the original post, but I’ll repeat it here for redundancy:

If you want to deny access to your Apache instance from a list of IP addresses, you can create a file in your conf.d called “global_deny.conf” that looks like this:

<DirectoryMatch .*>
Order allow,deny
allow from all
deny from […]
deny from […]
 <DirectoryMatch>

Note the “order allow,deny” and the initial “allow from all” lines. Those are very important. :)
